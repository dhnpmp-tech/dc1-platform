Motion & Animation NEW

🔧 For SPARK: This section defines all animation timings and easing functions. Motion communicates state, never decoration.

Core Principles

  • Functional: Motion clarifies state changes. Hover lifts. Loading spins. Transitions slide.
  • Fast: Vercel style — quick, snappy, invisible. Never slow/bouncy.
  • Accessible: Respect prefers-reduced-motion media query. Never force animations.
  • Consistent: Same timing across all interactions. Predictability builds trust.

Timing Scale

Duration Use Case Examples
150ms (Fast) Micro-interactions Button hover, checkbox toggle, menu open
200ms (Standard) UI state changes Tab switch, modal fade, tooltip appear
300ms (Slow) Page transitions Router navigation, sidebar collapse, overlay
500ms+ (Slower) Complex animations Scroll reveals, staggered list items

Easing Functions

// Vercel-style easing $easing-standard: cubic-bezier(0.4, 0, 0.2, 1) // Default smoothness $easing-out: cubic-bezier(0, 0, 0.2, 1) // Entrance (decelerating) $easing-in: cubic-bezier(0.4, 0, 1, 1) // Exit (accelerating) $easing-sharp: cubic-bezier(0.4, 0, 0.6, 1) // Quick state changes

Common Animations

Button Interaction

Hover: transform: translateY(-2px); box-shadow: 0 4px 12px
Duration: 150ms ease-out
Active: transform: translateY(0px); box-shadow: 0 2px 4px
Duration: 100ms ease-in

Loading Spinner

Animation: 360° rotation
Duration: 1s linear, infinite
Size: 24px
Color: #F5A524 (DCP Amber)

Page Transition

Enter: Fade in + slideUp (20px)
Duration: 300ms ease-out
Exit: Fade out + slideDown
Duration: 200ms ease-in

Accessibility: Respecting Motion Preferences

@media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }
⚠️ CRITICAL: Always include this rule. Users with vestibular disorders will experience nausea from animations if not respected.

Focus States

Focus Ring: 3px solid #4DB8E8 (DCP Link Blue)
Offset: 2px outside element
Timing: Instantly visible, no delay
Required: All interactive elements (buttons, links, inputs, dropdowns)

Keyboard Navigation

  • All interactive elements must be keyboard-accessible
  • Tab order must be logical (left-to-right, top-to-bottom)
  • Skip navigation links at top of page
  • Modal focus trap (tab stays within modal)
  • Escape key dismisses modals and dropdowns
  • Enter/Space activates buttons and checkboxes

Screen Reader Support

ARIA Labels (Critical)

// Icon-only buttons MUST have aria-label <button aria-label="Close modal">✕</button> // Form inputs MUST have associated labels <label for="gpu-model">GPU Model</label> <input id="gpu-model" type="text" /> // Status indicators MUST have aria-live <div aria-live="polite" aria-label="Provider status"> Online </div>

Color & Contrast

❌ Never: Convey information by color alone
  • ❌ Red background = error (need icon + text)
  • ❌ Green border = success (need checkmark + text)

✅ Always: Combine color + icon + text

Mobile-First Rules

  • Hide/collapse sidebar below lg breakpoint (show hamburger menu)
  • Single column on mobile, 2-column on tablet, 3-4 on desktop

Error & Edge States NEW

🔧 For SPARK & SYNC: Happy path is easy. These states make or break the product. Design them first.

Error Page (500)

Visual: Lightning bolt icon (muted/gray)
Headline: "Something went wrong on our end."
Action: "Refresh page" button + error code (support reference)
Tone: Apologetic, not scary

Not Found Page (404)

Visual: Cloud illustration (broken/glitched)
Headline: "This page doesn't exist."
Action: "Go home" link
Suggestion: Related links or sitemap

Empty State

When: Dashboard with no data (new provider, no jobs yet)
Visual: Relevant illustration (e.g., empty GPU rack)
Headline: "No jobs yet"
CTA: Clear next action ("Submit your first job")
Tone: Encouraging, not discouraging

Offline / Degraded Service

Banner placement: Top of page, sticky
Background: #ffc107 (warning amber)
Icon: WiFi-off icon
Message: "You're offline. Some data may be stale."
Duration: Stays until connection restored

Loading States

Skeleton Screen

Use for: Data that takes >200ms to load
Animate: Subtle pulse (1.5s cycle)
Color: #111133 shimmer over #090E1A
Don't use: "Loading..." text alone (skeleton is better UX)

Validation & Real-Time Feedback

Email field: Show error instantly on blur (red border + message)
Password strength: Show meter as user types
Success: Green checkmark when valid
Timing: Debounce 300ms to avoid flashing

Technical Terms (Stay English)

These don't translate well and are industry standard. Keep them in English even in Arabic text:

  • GPU, VRAM, CUDA, NVIDIA, RTX, H100, A100
  • API, REST, JSON, HTTP
  • Daemon, provider, renter, compute
  • Bitcoin, Ethereum, blockchain (if mentioned)

Arabic Voice & Tone

  • Register: Modern Standard Arabic (not colloquial/dialects)
  • Tone: Professional, direct, Saudi context
  • Translation style: Sense-for-sense (not word-for-word)
  • Example: "Infinite Compute. Real Power." → "القوة، رقميًا" (Al-qawwa, raqmiyyan)

RTL Layout Checklist

  • Logo/branding still left-to-right (universal)
  • Navigation reverses in RTL view
  • Icons that indicate direction are mirrored
  • Form inputs still work in RTL (text flows right-to-left)
  • Tables: headers RTL, data respects RTL flow
  • All interactive elements function identically

Implementation Guides

For Frontend Developers (SPARK)

  • Use semantic Tailwind class names (bg-accent, text-primary, not hardcoded hex)
  • Apply motion specs from Motion & Animation section
  • Implement all component states from Components & States
  • Enforce accessibility via focus rings and aria labels
  • Use mobile-first responsive breakpoints
  • Respect prefers-reduced-motion for animations

For QA & Testing (SYNC)

  • Test against responsive device matrix (5 devices minimum)
  • Run accessibility scans (axe, Pa11y) on all pages
  • Validate contrast ratios with contrast checker
  • Test keyboard navigation (Tab, Enter, Escape)
  • Verify error states are designed and functional
  • Test motion on low-end devices (Galaxy A14 proxy)
  • Validate all touch targets are 44×44px minimum

For Marketing (VOLT)

  • Use Figma templates for all social posts
  • Maintain brand voice consistency (direct, technical, Saudi-first)

  • Use OG image dimensions exactly (no scaling/stretching)
  • Phase Arabic rollout: Phase 1 = hero/pricing/CTAs, Phase 2 = full platform
  • Include Arabic speaker in translation review (not just machine translation)

For Compliance & Security (GUARDIAN)

  • Verify dark + light mode toggle functionality
  • Ensure data residency messaging appears in hero section
  • Audit privacy policy against PDPL requirements
  • Review Arabic translations for regulatory accuracy (SAMA/CITC terms)
  • Validate WCAG AAA compliance before launch
  • Test RTL layout on all key pages
