## Goal

Complete a workload flow:

1. Validate your renter key.
2. Find an online provider.
3. Submit a workload.
4. Poll status until completion.
5. Fetch final output.

This guide intentionally avoids payment/top-up steps and focuses on pure compute lifecycle APIs.

## Why teams choose DCP

- **Saudi energy-cost advantage**: workloads run in a structurally lower electricity-cost market for better long-run compute economics.
- **Arabic AI model support**: production paths for **ALLaM 7B**, **Falcon H1**, **JAIS 13B**, and **BGE-M3**.
- **Container-based execution**: bring your own container workloads with NVIDIA-accelerated Docker runtime.

## 1) Set environment variables

```bash
export DCP_API="https://dcp.sa/api/dc1"
export RENTER_KEY="dcp-renter-your-key"
```

## 2) Validate account access

```bash
curl -s "$DCP_API/renters/me?key=$RENTER_KEY"
```

## 3) Discover available providers

```bash
curl -s "$DCP_API/renters/available-providers"
```

Pick a `provider.id` from the response and export it:

```bash
export PROVIDER_ID="42"
```

## 4) Submit a workload

```bash
JOB_JSON=$(curl -s -X POST "$DCP_API/jobs/submit" \
  -H "Content-Type: application/json" \
  -H "x-renter-key: $RENTER_KEY" \
  -d '{
    "provider_id": '"$PROVIDER_ID"',
    "job_type": "llm_inference",
    "duration_minutes": 3,
    "max_duration_seconds": 180,
    "container_spec": {
      "image_type": "vllm-serve"
    },
    "params": {
      "model": "TinyLlama/TinyLlama-1.1B-Chat-v1.0",
      "prompt": "Write 3 bullets about AI in Riyadh"
    }
  }')

echo "$JOB_JSON"
JOB_ID=$(echo "$JOB_JSON" | jq -r '.job.job_id // .job_id')
echo "JOB_ID=$JOB_ID"
```

## 5) Poll status and fetch output

```bash
while true; do
  status=$(curl -s -H "x-renter-key: $RENTER_KEY" "$DCP_API/jobs/$JOB_ID" | jq -r '.status')
  echo "status=$status"
  if [ "$status" = "completed" ]; then
    break
  fi
  if [ "$status" = "failed" ] || [ "$status" = "cancelled" ]; then
    echo "Job ended before completion: $status"
    break
  fi
  sleep 5
done

curl -s "$DCP_API/jobs/$JOB_ID/output?key=$RENTER_KEY"
```

Before completion, status will remain `pending`, `queued`, or `running`.
