## الهدف

إنهاء مسار إرسال مهمة بشكل مباشر:

1. التحقق من مفتاح المستأجر.
2. إرسال مهمة.
3. متابعة الحالة أو بث السجلات.
4. جلب المخرجات النهائية.

## 1) ضبط متغيرات البيئة

```bash
export DCP_API="https://dcp.sa/api/dc1"
export RENTER_KEY="dcp-renter-your-key"
```

## 2) التحقق من الحساب

```bash
curl -s "$DCP_API/renters/me?key=$RENTER_KEY"
```

## 3) إرسال المهمة

```bash
JOB_JSON=$(curl -s -X POST "$DCP_API/jobs/submit" \
  -H "Content-Type: application/json" \
  -H "x-renter-key: $RENTER_KEY" \
  -d '{
    "provider_id": 42,
    "job_type": "llm-inference",
    "duration_minutes": 3,
    "max_duration_seconds": 180,
    "container_spec": {
      "image_type": "vllm-serve"
    },
    "params": {
      "model": "TinyLlama/TinyLlama-1.1B-Chat-v1.0",
      "prompt": "اكتب 3 نقاط عن الذكاء الاصطناعي في الرياض"
    }
  }')

echo "$JOB_JSON"
JOB_ID=$(echo "$JOB_JSON" | jq -r '.job.job_id // .job_id')
echo "JOB_ID=$JOB_ID"
```

## 4) بث السجلات (اختياري)

```bash
curl -N "$DCP_API/jobs/$JOB_ID/logs/stream?key=$RENTER_KEY"
```

## 5) جلب المخرجات

```bash
curl -s "$DCP_API/jobs/$JOB_ID/output?key=$RENTER_KEY"
```

إذا لم تنتهِ المهمة بعد، قد تحصل على `202` حتى الوصول للحالة النهائية.
