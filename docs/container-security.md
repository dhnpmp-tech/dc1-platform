# Container Security Baseline

This document defines the runtime hardening enforced for provider-side Docker job execution in DCP.

## Scope

- `backend/installers/dc1_daemon.py`
- `backend/installers/dc1-daemon.py` (fallback copy)
- `backend/src/services/docker-manager.ts` (Mission Control wrapper path)

## Enforced Controls

### 1) No Privilege Escalation

- `--security-opt no-new-privileges:true`
- `--cap-drop all`

### 2) Resource Limits Per Job

Script workload containers (`run_docker_job`) enforce per-job-type limits:

- `--cpus`
- `--memory`
- `--memory-swap` (equal to memory, swap headroom disabled)
- `--pids-limit`
- `--shm-size`

vLLM serve containers (`run_vllm_serve_job`) enforce model-tier limits (small vs large models) using the same controls.

### 3) Read-only Root Filesystem Where Possible

For script execution containers:

- `--read-only`
- writable scratch only via tmpfs:
  - `--tmpfs /tmp:rw,noexec,nosuid,size=...`
  - `--tmpfs /var/tmp:rw,noexec,nosuid,size=256m`

For vLLM serve containers, rootfs remains writable because model download/caching is required at startup.

### 4) Seccomp Profile

- A dedicated seccomp policy is generated by daemon (`/tmp/dc1-gpu-seccomp.json`)
- Applied when available via `--security-opt seccomp=/tmp/dc1-gpu-seccomp.json`
- If profile generation fails, Docker default seccomp profile is used

### 5) Network Segmentation

- Script execution containers: `--network none`
- vLLM serve containers: `--network bridge` only because API endpoint publishing requires `-p host:8000`

## Verification

Run these checks on a provider machine after starting a job container:

```bash
docker inspect <container_id> --format '{{json .HostConfig.SecurityOpt}}'
docker inspect <container_id> --format '{{.HostConfig.Memory}} {{.HostConfig.MemorySwap}} {{.HostConfig.NanoCpus}} {{.HostConfig.PidsLimit}}'
docker inspect <container_id> --format '{{.HostConfig.ReadonlyRootfs}} {{json .HostConfig.Tmpfs}}'
docker inspect <container_id> --format '{{json .HostConfig.CapDrop}} {{.HostConfig.NetworkMode}}'
```

Expected:

- `SecurityOpt` includes `no-new-privileges:true`
- Memory/CPU/PIDs are non-zero and bounded
- Script containers show `ReadonlyRootfs=true`
- Capabilities include dropped `ALL`
- Network mode is `none` for script jobs
