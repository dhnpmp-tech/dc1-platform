## 📋 Section 4: Analysis Frameworks

### 4.1 Issue Severity Classification

```
CRITICAL (blocks launch):
- Task cannot be completed by 70%+ of users
- Safety/security issue
- Data loss or payment error possible

HIGH (fix before launch):
- Task takes 2x+ expected time
- Copy is misleading
- Icon/label unclear to majority (>50%)
- Top 3 user pain points

MEDIUM (can fix post-launch):
- Affects <30% of users
- Minor usability friction
- Enhancement that improves experience
- Non-critical missing feature

LOW (backlog):
- Nice-to-have feature
- Advanced user workflow
- Accessibility improvement
- Rare edge case

Classification Template:
```
Issue: [Description]
Affected users: [#/N]
Frequency: [1/8, 3/8, etc.]
Impact: [User can't complete / adds 2+ min delay / confusing but workable / nice-to-have]
Root cause: [Why did this happen?]
Severity: [ ] Critical [ ] High [ ] Medium [ ] Low
Fix effort: [Hours estimate]
Recommended fix: [Specific action]
```
